Overview

This project shows Splunk Enterprise installed on a Windows Server Core VM running in VirtualBox. It covers unlocking the Server Core console, mounting a shared folder, running the MSI installer, and verifying Splunk directories (C:\\Program Files\\Splunk), plus navigating to etc for configuration. All screenshots and CLI outputs are captured from the live lab.

  • Hypervisor: VirtualBox
  • Guest: Windows Server (Server Core)
  • Role: Splunk Enterprise server
  • Focus: install, filesystem layout, basic service checks

Step-by-step

  1. Unlock Server Core: Use VirtualBox menu Input → Keyboard → Insert Ctrl+Alt+Del to log in.
  2. Confirm shared folder: net use and navigate to \\\\VBoxSvr\\Windows_Shared_Folder.
  3. Run MSI: start splunk-10.x.x-windows-x64.msi from the shared folder.
  4. Verify install: Check C:\\Program Files\\Splunk with ls. Expect bin, etc, var, etc.
  5. Explore config: cd \"C:\\Program Files\\Splunk\\etc\" to see apps, system, users.

Notes & Gotchas

  • Paths with spaces require quotes in PowerShell: cd \"program files\".
  • If net use shows the share but it’s empty, check your host-side folder permissions and that Auto-mount is configured in VirtualBox.
  • Server Core has no GUI for Splunk Web. Access Splunk Web from a browser on your host using the VM’s IP and port 8000, or manage Splunk with CLI commands in bin.
